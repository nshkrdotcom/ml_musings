That sentence means: **researchers want to preserve what makes backprop so effective, while removing or reducing its major weaknesses.**

The phrase has seven pieces.

# 1. “Keep the power of backprop”

Backprop is powerful because it gives an exact, efficient gradient for every trainable parameter in a neural network. Your docset’s four equations show this core mechanism: compute output-layer error, propagate it backward, then compute bias and weight gradients from the local error signal (\delta^l). 

In plain English:

```text
Backprop tells every weight:
“How much did you contribute to the final loss,
and in what direction should you change?”
```

Its power comes from three things:

| Strength                            | Meaning                                                    |
| ----------------------------------- | ---------------------------------------------------------- |
| Efficient                           | One forward pass + one backward pass gives all gradients   |
| Exact, within the computation graph | It applies the chain rule systematically                   |
| Scalable                            | Works for networks with millions or billions of parameters |

So “keep the power” means: do not throw away the ability to assign credit precisely across many layers.

# 2. “Make it cheaper”

Backprop is effective, but expensive.

Training requires:

```text
forward pass
+ storing intermediate activations
+ backward pass
+ gradient updates
```

The backward pass often costs about as much as, or more than, the forward pass. For huge models, this is a major bottleneck.

“Cheaper” can mean cheaper in several ways:

| Kind of cost  | What researchers try to reduce                          |
| ------------- | ------------------------------------------------------- |
| Compute       | Fewer matrix multiplications, fewer backward operations |
| Memory        | Store fewer activations                                 |
| Energy        | Use less GPU/TPU power                                  |
| Communication | Reduce gradient synchronization across devices          |
| Time          | Train faster                                            |

Examples:

* gradient checkpointing;
* low-rank adaptation like LoRA;
* parameter-efficient fine-tuning;
* sparse backprop;
* quantized gradients;
* local losses;
* partial freezing;
* zeroth-order or forward-only approximations.

The goal is not necessarily to beat exact backprop mathematically. The goal is to get **most of its benefit at lower cost**.

# 3. “Make it more local”

Standard backprop is not very local.

A weight in an early layer updates using information that came all the way from the final loss. That means early layers depend on a long backward chain.

In standard backprop:

```text
Layer 1 waits for Layer 2
Layer 2 waits for Layer 3
Layer 3 waits for Layer 4
...
Output loss sends signal backward
```

That creates what is sometimes called **backward locking**: earlier layers cannot finish their update until later layers have completed their backward computations.

“More local” means each layer or block could learn using mostly nearby information.

Instead of:

```text
global loss → exact backward chain → all layers
```

researchers explore:

```text
local loss at each layer
local prediction error
local target
local feedback signal
neighboring-layer correction
```

Examples:

| Method              | Locality idea                                                      |
| ------------------- | ------------------------------------------------------------------ |
| Local losses        | Each block has its own auxiliary objective                         |
| Target propagation  | Send desired activations backward, not exact gradients             |
| Predictive coding   | Layers reduce local prediction errors                              |
| Feedback alignment  | Use approximate/random feedback instead of exact transpose weights |
| Synthetic gradients | Predict the gradient locally instead of waiting for the true one   |

The big idea:

> Backprop is globally coordinated. Local learning tries to make each part of the network more independently trainable.

# 4. “Make it more biologically plausible”

This asks:

> Could something like backprop happen in brains?

Standard backprop is biologically questionable because it seems to require things real neurons probably do not have.

Problems include:

| Backprop requirement                 | Biological concern                                                                  |
| ------------------------------------ | ----------------------------------------------------------------------------------- |
| Exact symmetric feedback weights     | Brain feedback pathways are not exact transposes of forward pathways                |
| Global loss signal                   | Neurons usually receive local chemical/electrical signals                           |
| Stored forward activations           | Biological memory does not look like PyTorch activation buffers                     |
| Separate forward and backward phases | Brains operate continuously and asynchronously                                      |
| Precise derivatives                  | Spiking neurons and dendrites do not obviously compute exact chain-rule derivatives |

So “more biologically plausible” means finding learning rules that still solve credit assignment but look more like neural systems.

Examples:

* Hebbian learning: “neurons that fire together wire together”;
* spike-timing-dependent plasticity;
* predictive coding;
* feedback alignment;
* equilibrium propagation;
* dendritic error signals;
* eligibility traces;
* neuromodulated reward learning.

Important caveat: biologically plausible does **not** automatically mean better for ML. It means the method might teach us something about brains, or inspire more local, parallel, energy-efficient AI hardware.

# 5. “Make it more memory-efficient”

This is one of the most practical issues.

During the forward pass, backprop stores intermediate values:

```text
z¹, a¹, z², a², z³, a³, ...
```

Why? Because the backward pass needs them.

For example, the weight gradient:

[
\frac{\partial C}{\partial W^l} = \delta^l(a^{l-1})^T
]

requires (a^{l-1}), the activation from the forward pass. Your docset correctly highlights that the weight gradient depends on the incoming activation and the local error signal. 

For large models, storing all activations can be enormous.

Memory-saving approaches include:

| Technique                | Idea                                               |
| ------------------------ | -------------------------------------------------- |
| Gradient checkpointing   | Store only some activations; recompute the rest    |
| Reversible layers        | Reconstruct earlier activations from later ones    |
| Activation compression   | Store activations in lower precision               |
| Offloading               | Move activations to CPU/storage temporarily        |
| Frozen layers            | Do not store gradients for parts not being trained |
| LoRA/adapters            | Train small modules instead of full model          |
| Implicit differentiation | Avoid storing every step of an iterative process   |

Tradeoff:

```text
less memory often means more recomputation
```

So memory-efficient backprop often exchanges memory for extra compute.

# 6. “Make it more stable through time”

This mostly refers to recurrent networks, sequence models, dynamical systems, reinforcement learning, neural ODEs, and any model where computation unfolds across many steps.

For recurrent models, backprop becomes **backpropagation through time**:

```text
t=1 → t=2 → t=3 → ... → t=T
```

Then the backward pass goes:

```text
T → T-1 → T-2 → ... → 1
```

The problem is repeated Jacobian multiplication.

If derivatives are usually smaller than 1:

```text
gradient vanishes
```

If derivatives are usually larger than 1:

```text
gradient explodes
```

So “more stable through time” means improving long-range credit assignment.

Examples:

| Method                                 | Stability benefit                                |
| -------------------------------------- | ------------------------------------------------ |
| LSTM/GRU gates                         | Preserve information over longer time spans      |
| Residual connections                   | Give gradients shorter paths                     |
| Gradient clipping                      | Prevent exploding gradients                      |
| Orthogonal initialization              | Preserve signal magnitude                        |
| Normalization                          | Stabilize activation distributions               |
| Truncated BPTT                         | Avoid extremely long backward chains             |
| Eligibility traces                     | Approximate long-term credit locally             |
| Implicit/recurrent equilibrium methods | Differentiate final state rather than every step |

The issue is not just memory. It is also mathematical stability.

# 7. “Better suited to exotic computation graphs”

Classic backprop was explained for ordinary feedforward neural nets:

```text
input → hidden layers → output → loss
```

But modern AI differentiates through much stranger systems:

```text
neural network + simulator
neural network + search algorithm
neural network + physics engine
neural network + renderer
neural network + optimizer
neural network + ODE solver
neural network + sampling process
neural network + discrete decisions
```

These are “exotic computation graphs.”

Examples:

| Exotic graph           | Why backprop gets tricky                                        |
| ---------------------- | --------------------------------------------------------------- |
| Neural ODEs            | Need gradients through continuous-time solvers                  |
| Differentiable physics | Collisions/contact can be non-smooth                            |
| Rendering              | Visibility and occlusion can be discontinuous                   |
| Reinforcement learning | Rewards are delayed and noisy                                   |
| Probabilistic programs | Random sampling breaks ordinary gradients                       |
| Discrete choices       | Argmax, sampling, and branching are not smoothly differentiable |
| Optimization layers    | Need to differentiate through another optimizer                 |
| Equilibrium models     | Output is a fixed point, not a normal layer stack               |

So the phrase means:

> Backprop needs to evolve from “differentiate through a stack of layers” into “differentiate through complicated computational processes.”

# The whole sentence, translated

Original:

> Keep the power of backprop, but make it cheaper, more local, more biologically plausible, more memory-efficient, more stable through time, and better suited to exotic computation graphs.

Plain version:

> We want learning algorithms that still assign credit as effectively as backprop, but that use less compute and memory, rely less on exact global backward signals, look more like plausible biological learning, work better over long sequences, and can handle complicated systems beyond ordinary neural-network layers.

# One-line version

Backprop is still the champion because it solves credit assignment well; the frontier is making that credit assignment **cheaper, more distributed, more stable, more biologically realistic, and usable in stranger computational worlds.**
