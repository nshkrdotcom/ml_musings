# Crash Course 5: SVD, Rank, and LoRA — Compressing Changes Instead of Relearning Everything

This lesson maps mainly to:

```text
08_lora_and_svd.exs
09_non_redundant_compression.exs
```

The big idea:

> Many useful changes to a model are much smaller than the full model itself.

Instead of retraining every weight, LoRA learns a small low-rank “update path” that nudges the model.

---

# 1. Start with a weight matrix

A neural network layer often does this:

```text
output = input × weights
```

The weights are stored in a matrix.

Example:

```text
W₀ = original frozen weight matrix
```

You can think of `W₀` as the model’s existing skill.

It already knows a lot. We do not want to destroy it.

---

# 2. What does rank mean?

Rank tells you how many truly independent directions a matrix contains.

A matrix can look large but still be redundant.

Example:

```text
[1  2  3  4]
[2  4  6  8]
[3  6  9 12]
[4  8 12 16]
```

Rows 2, 3, and 4 are just scaled versions of row 1.

So this matrix looks 4D, but it really has only one independent direction.

That is a **rank-1 matrix**.

The key intuition:

> Rank measures independent information, not raw size.

---

# 3. SVD: finding the hidden directions

SVD stands for **Singular Value Decomposition**.

It breaks a matrix into:

```text
W = U Σ Vᵀ
```

Think of that as:

```text
U  = output directions
Σ  = strength of each direction
Vᵀ = input directions
```

The singular values in `Σ` tell you how important each direction is.

Large singular value:

```text
important direction
```

Tiny or zero singular value:

```text
weak or redundant direction
```

So SVD is like an X-ray for matrices.

It shows how much real structure is inside.

---

# 4. Compression with SVD

If a matrix has mostly redundant directions, we can keep only the strongest ones.

Instead of keeping all directions:

```text
direction 1
direction 2
direction 3
direction 4
```

we keep:

```text
direction 1
```

If the other directions carry almost no information, the reconstructed matrix is nearly the same.

That is low-rank compression.

The lesson script `08_lora_and_svd.exs` uses a deliberately redundant matrix, so rank-1 compression works almost perfectly.

Then `09_non_redundant_compression.exs` shows the opposite case: if a matrix is truly full-rank, forcing it into rank 1 destroys information.

That contrast matters.

Low-rank compression works when the useful structure is actually low-rank.

---

# 5. Why this matters for model adaptation

Suppose you have a large pretrained model.

Its original weights are:

```text
W₀
```

You want to adapt it to a new task.

Naively, you could train all of `W₀`.

But that is expensive and risky.

LoRA asks:

> What if the needed change is low-rank?

Instead of replacing the whole matrix, LoRA adds a small update:

```text
W = W₀ + ΔW
```

The original model stays frozen.

Only the update `ΔW` is trained.

---

# 6. The LoRA trick

LoRA represents the update as two smaller matrices:

```text
ΔW = B A
```

Where:

```text
A = down-projection
B = up-projection
```

So instead of learning one huge matrix:

```text
D × D
```

LoRA learns two smaller matrices:

```text
r × D
D × r
```

where `r` is small.

For example, if:

```text
D = 4096
r = 8
```

then full update size is:

```text
4096 × 4096 = 16,777,216 parameters
```

LoRA update size is roughly:

```text
8 × 4096 + 4096 × 8 = 65,536 parameters
```

That is tiny compared to the full matrix.

---

# 7. The path through LoRA

For an input vector `x`, the frozen model does:

```text
x → W₀ → original output
```

LoRA adds a side path:

```text
x → A → small compressed space → B → update output
```

Then the two outputs are added:

```text
final output = frozen output + LoRA update
```

So the model keeps its old knowledge, but gains a small adjustable steering layer.

---

# 8. Why this is powerful

LoRA works because many task-specific adaptations do not require changing every direction in the model.

They often need to adjust a small number of meaningful axes.

For example:

```text
more legal style
better medical formatting
stronger code completion
more concise answers
domain vocabulary
```

Those changes can often be represented as a low-rank update.

Not always. But often enough to be very useful.

---

# 9. The important distinction

SVD and LoRA are related, but not identical.

**SVD** analyzes or compresses an existing matrix.

**LoRA** trains a new low-rank update.

So:

```text
SVD = discover existing important directions
LoRA = learn a small new directional patch
```

Both rely on the idea that useful structure may live in fewer dimensions than the full matrix size suggests.

---

# 10. The sticky mental model

Imagine `W₀` is a huge sound mixing board with millions of knobs.

Full fine-tuning says:

```text
adjust every knob
```

LoRA says:

```text
leave the board alone
add a small side controller
let the side controller nudge only the important patterns
```

That is the spirit.
